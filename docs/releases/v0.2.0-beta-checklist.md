# v0.2.0-beta.0 Release Checklist

**Target Release**: v0.2.0-beta.0  
**Prerequisites**: All [beta gate criteria](./v0.2.0-beta-gate.md) must pass

## Pre-Beta Validation

### Gate Verification
- [ ] All items in [v0.2.0-beta-gate.md](./v0.2.0-beta-gate.md) checked off
- [ ] CI green for 3 consecutive runs on `develop` branch
- [ ] No open P0/P1 issues with `v0.2.0` milestone

### CI Jobs (Must Pass)
- [ ] `navigation-tests` – Nav/core, golden tests
- [ ] `test` – Full test suite (Linux, macOS, Windows)
- [ ] `ratatui-testlib` – Smoke subset
- [ ] `lint` – fmt + clippy clean
- [ ] `build-release` – All platform builds succeed
- [ ] `security` – cargo audit clean
- [ ] `docs` – rustdoc + mdBook build

### Documentation Audit
- [ ] `docs/README.md` index complete with valid links
- [ ] Deprecated docs have banners pointing to replacements
- [ ] mdBook portal builds: `mdbook build docs/book`
- [ ] External site URL documented (if deployed)
- [ ] CHANGELOG.md has v0.2.0-beta.0 section drafted

### API/ABI Freeze Verification
- [ ] Plugin ABI reviewed – no breaking changes since alpha freeze
- [ ] Keymap/config documented in mdBook
- [ ] Navigation API stable – no planned changes

### Telemetry Counters
- [ ] Navigation counters implemented and tested
- [ ] Render counters implemented and tested  
- [ ] Image counters implemented and tested
- [ ] Counter documentation in TELEMETRY.md

## Release Execution

### Version Bump
- [ ] Update workspace `Cargo.toml` version to `0.2.0-beta.0`
- [ ] Run `cargo check --workspace` to verify
- [ ] Update VERSION file if present

### CHANGELOG Update
- [ ] Add `## v0.2.0-beta.0 - YYYY-MM-DD` section
- [ ] List all changes since v0.2.0-alpha.X
- [ ] Document known issues
- [ ] Note API freeze commitment

### Git Operations
```bash
# Create release commit
git add -A
git commit -m "chore: prepare v0.2.0-beta.0 release"

# Create annotated tag
git tag -a v0.2.0-beta.0 -m "Release v0.2.0-beta.0

Beta release with frozen API/ABI.
See CHANGELOG.md for details."

# Push
git push origin develop
git push origin v0.2.0-beta.0
```

### GitHub Release
- [ ] Create release from tag on GitHub
- [ ] Copy CHANGELOG section to release notes
- [ ] Mark as **pre-release** (beta)
- [ ] Attach binary artifacts once CI completes

## Post-Release

### Verification
- [ ] GitHub Release published successfully
- [ ] CI release workflow completed
- [ ] Binary downloads work on all platforms
- [ ] mdBook documentation accessible

### Communication
- [ ] Announce in GitHub Discussions
- [ ] Update project status in README if needed
- [ ] Notify beta testers/community

### Tracking
- [ ] Create v0.2.0-beta.1 milestone for bug fixes
- [ ] Create v0.2.0 milestone for stable release
- [ ] Schedule stabilization period review (2 weeks)

## Rollback Plan

If critical issues discovered after beta tag:

1. **Do NOT delete the tag** – maintain audit trail
2. Create hotfix on release branch
3. Tag v0.2.0-beta.1 with fix
4. Update GitHub Release notes with advisory
